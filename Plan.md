# AHK Manager — GUI App Plan

## Overview

A desktop GUI application for Windows that lets users create, manage, and apply AutoHotKey script profiles without manually editing `.ahk` files. Users can define hotkeys, group them into named profiles, switch between profiles, and run/stop the active script — all from a clean interface.

---

## Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Language | Python 3.11+ | Fast to build, good AHK subprocess control |
| GUI | CustomTkinter | Modern flat look on top of tkinter, no extra install pain |
| Persistence | JSON (local) | Simple; one file per profile or a single `profiles.json` |
| AHK Execution | `subprocess` | Launch/kill `AutoHotkey.exe` with a generated `.ahk` file |
| Packaging | PyInstaller | Single `.exe` for distribution |

---

## Feature Set

### 1. Profile Management
- Create a new named profile
- Rename / duplicate an existing profile
- Delete a profile (with confirmation)
- Active profile shown prominently; only one profile runs at a time

### 2. Hotkey Editor
Each hotkey entry has:
- **Trigger** — key combo (e.g. `Ctrl+Shift+4`), entered via a recorder or typed manually
- **Action type** — one of:
  - `Send Text` — types a string, with optional `{Enter}` / `{Tab}` appended
  - `Run Command` — runs a program or shell command
  - `Window: Always on Top` — toggle for the active window
  - `Custom AHK` — raw AHK snippet for power users
- **Label / Description** — optional human-readable note
- **Enabled toggle** — disable a hotkey without deleting it

CRUD operations: Add, Edit (double-click row), Delete, Reorder (drag).

### 3. Apply / Run
- **Apply** button: generates a temp `.ahk` file from the active profile and launches `AutoHotkey.exe`
- **Stop** button: terminates the running AHK process
- **Status indicator**: shows Running / Stopped with a colored dot
- If a profile is already running and the user edits it, prompt "Restart to apply changes?"

### 4. Save / Import / Export
- Auto-save to `profiles.json` on every edit (no explicit Save needed)
- **Export Profile → .ahk**: writes a clean, commented `.ahk` file the user can use standalone
- **Import .ahk**: parses a simple `.ahk` file and creates a new profile from it (best-effort for standard `Send` / `Run` hotkeys)

### 5. Settings
- Path to `AutoHotkey.exe` (auto-detected; user can override)
- Default profile to apply on app launch
- Start minimized to system tray option (stretch goal)

## Data Model

```json
// profiles.json
{
  "active_profile": "work-macros",
  "profiles": {
    "work-macros": {
      "name": "Work Macros",
      "hotkeys": [
        {
          "id": "uuid",
          "trigger": "^+4",
          "action_type": "send_text",
          "action_value": "claude --dangerously-skip-permissions",
          "append_enter": true,
          "description": "Claude CLI command",
          "enabled": true
        },
        {
          "id": "uuid",
          "trigger": "^SPACE",
          "action_type": "always_on_top",
          "action_value": "",
          "append_enter": false,
          "description": "Toggle always on top",
          "enabled": true
        }
      ]
    }
  },
  "settings": {
    "ahk_exe_path": "C:/Program Files/AutoHotkey/AutoHotkey.exe",
    "launch_profile_on_start": null,
    "start_minimized": false
  }
}
```

## AHK Code Generation

The app generates a `.ahk` file from the profile data at runtime:

```
; Generated by AHK Manager — Profile: Work Macros
; Do not edit manually — use AHK Manager instead

^+4::  ; Claude CLI command
    Send, claude --dangerously-skip-permissions
    Send, {Enter}
return

^SPACE::  ; Toggle always on top
    Winset, Alwaysontop, , A
return
```

## Dependencies

```
customtkinter>=5.2
pyinstaller>=6.0   # packaging only
```

No AHK Python bindings needed — the app shells out to `AutoHotkey.exe` directly.

---

## Notes

- Generated `.ahk` is written to a temp file (`%TEMP%\ahk_manager_active.ahk`) so it doesn't pollute the project folder.
- Only one AHK process is managed at a time; switching profiles kills the old process and starts a new one.
- The existing `Untitled.ahk` will be parsed and imported as the initial "Default" profile on first run.
